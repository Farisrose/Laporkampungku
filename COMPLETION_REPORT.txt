   - Server maintains state via $_SESSION
   - Verified on each protected page access

   - One-way hashing (cannot be reversed)
   - Slow algorithm (prevents brute force)
   - Random salt per password
   - Industry standard for password storage

   - 4 user levels with different permissions
   - UI rendered based on user.level
   - Sidebar menu dynamically generated
   - Scalable for adding new roles

4. Frontend Form Handling
   - preventDefault() to avoid page reload
   - Fetch API for AJAX requests
   - JSON response parsing
   - Error handling and user feedback
   - Role-based redirects

5. Database Design
   - 8 tables with proper relationships
   - Foreign keys for referential integrity
   - Prepared statements for security
   - Audit trail for tracking changes
